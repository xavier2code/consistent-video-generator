# 序列视频生成功能说明

## 功能概述

序列视频生成功能允许上传**2-6张图片**，自动生成对应数量的视频片段并合并成一个完整的视频。

**动态生成规则：**
- 上传 **n 张图片** → 生成 **n-1 个视频**
- 例如：
  - 2张图片 → 1个视频（无需合并）
  - 3张图片 → 2个视频 → 合并成1个
  - 4张图片 → 3个视频 → 合并成1个
  - 6张图片 → 5个视频 → 合并成1个

## API接口

### 生成序列视频

**端点:** `POST /api/v1/generate-sequence`

**参数:**
- `files`: 2-6张图片文件（必需，按顺序上传）
- `prompt`: 视频生成提示词（可选）

**工作流程:**

1. **上传2-6张图片** - 按时间顺序排列的人物照片
2. **动态生成视频片段:**
   - 根据上传的图片数量自动计算视频数量
   - 例如4张图片：
     - 视频1: 图片1 → 图片2
     - 视频2: 图片2 → 图片3
     - 视频3: 图片3 → 图片4
3. **自动下载** - 等待所有视频生成完成并下载到本地
4. **视频合并** - 使用ffmpeg将所有片段拼接成完整视频（单个视频则跳过）
5. **返回结果** - 返回合并后的视频URL

**响应示例:**
```json
{
  "task_id": "seq_a1b2c3d4e5f6g7h8",
  "status": "completed",
  "message": "序列视频生成并合并完成（4张图片 → 3个视频）",
  "total_videos": 3,
  "processed_videos": 3,
  "merged_video_url": "http://localhost:8000/videos/seq_a1b2c3d4e5f6g7h8_merged.mp4"
}
```

## 使用示例

### 示例1: 上传4张图片

```bash
curl -X POST "http://localhost:8000/api/v1/generate-sequence" \
  -F "files=@photo1.jpg" \
  -F "files=@photo2.jpg" \
  -F "files=@photo3.jpg" \
  -F "files=@photo4.jpg" \
  -F "prompt=同一人物在不同时期的平滑过渡"
```

**结果:** 生成3个视频并合并
